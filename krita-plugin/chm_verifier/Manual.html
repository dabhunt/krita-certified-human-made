Certified Human-Made (CHM)

Version: 1.0.0-alpha
Purpose: Cryptographically prove your artwork is human-made

Introduction

In a world where AI can generate photorealistic art in seconds, human creativity needs authentication. This plugin gives artists a tool to provide undeniable proof your work is 100% human-made.

What Is This?

The Certified Human-Made is a Krita plugin that captures your creative process as you work, generating cryptographic proof that your artwork was made by a human, not AI.

How It Works

  1. Real-time Capture: Monitors your brush strokes, layers, and workflow as you create
  2. Privacy-First Analysis: All processing happens locally on your computer
  3. Smart Classification: Detects pure human work vs. AI-assisted vs. mixed media
  4. Immutable Timestamp: Third-party verified timestamps via GitHub Gist's cryptographic commit history
  5. Verifiable Certificates: Share proofs that anyone can verify

Classification Types

Type Description Example
Human-Made Purely human work. No AI plugins detected, no imported images visible in final export. Drawing from imagination, life, or using hidden reference images for anatomy/pose study
Mixed Media Non-reference images imported (may be visible in final export) Photo collage with digital painting overlay
AI-Assisted AI plugins detected and enabled during creation Stable Diffusion or AI generation tools used

Important Note: Reference Images

Using reference images is a normal and professional part of the artistic process. Hidden reference images used for anatomy study, pose reference, or color inspiration maintain the Human-Made classification. The system only classifies as Mixed Media when non-reference images are imported and may be visible in the final export.

Using the Plugin

Starting a Session

When you create or open a document in Krita, the plugin automatically begins monitoring your work.

Checking Status

Open Tools → Scripts → Scripter to see debug messages confirming the plugin is active. You should see messages like:

  • CHM: Setup called
  • CHM: Loaded CHM library version 0.1.0
  • CHM: Event capture started

Key Features

For Artists:

  • ✅ One-click certification after finishing artwork
  • ✅ Privacy-protected: Your data never leaves your computer
  • ✅ Transparent: See exactly what's being verified
  • ✅ No account required for local verification

For Verifiers (Commissioners, Galleries):

  • ✅ Cryptographic proof: ED25519 signatures prevent tampering
  • ✅ GitHub Gist timestamp: Immutable, third-party verified proof-of-existence
  • ✅ Detailed classification: Know if work is pure human, referenced, etc.
  • ✅ Open verification: Anyone can validate without special software

Privacy & Security

  • Local-first: All analysis happens on your computer
  • Encrypted: Session data uses AES-256-GCM encryption
  • No tracking: We don't collect any personal data
  • Open source: All code is publicly auditable on GitHub

Technical Stack

  • Rust Core: Performance and security
  • Python Plugin: Krita integration
  • Immutable Timestamp: GitHub Gist (public, third-party verified) and CHM Local Log (HMAC-signed)
  • Cryptography: AES-256-GCM, ED25519, SHA-256

Troubleshooting

Plugin not appearing in list?

  • Verify the .desktop file is in the correct location (pykrita root, not inside plugin folder)
  • Restart Krita completely

"CHM library not available" error?

  • Rebuild the Rust library: cargo build --release
  • Copy to plugin: cp target/release/libchm.dylib krita-plugin/chm_verifier/lib/chm.so
  • Restart Krita

No debug messages in Scripter?

  • Make sure the plugin is enabled in Settings → Configure Krita → Python Plugin Manager
  • Restart Krita after enabling
  • Open Scripter console before creating/opening a document
