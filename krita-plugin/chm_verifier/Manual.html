Certified Human-Made (CHM) Verifier

Version: 0.1.0-alpha
Purpose: Cryptographically prove your artwork is human-made

What Is This?

The Certified Human-Made Verifier is a Krita plugin that captures your creative process as you work, generating cryptographic proof that your artwork was made by a human, not AI.

How It Works

  1. Real-time Capture: Monitors your brush strokes, layers, and workflow as you create
  2. Privacy-First Analysis: All processing happens locally on your computer
  3. Smart Classification: Detects pure human work vs. AI-assisted vs. traced
  4. Triple Timestamp Verification: Immutable timestamps via GitHub, Internet Archive, and CHM Public Log
  5. Verifiable Certificates: Share proofs that anyone can verify

Classification Types

Type Description Example
Pure Human-Made No AI assistance, no tracing Drawing from imagination or life
Referenced Uses reference images but hand-drawn Photo reference for anatomy study
AI-Assisted AI plugins used for generation/editing Stable Diffusion for backgrounds
Traced Direct tracing over imported images >70% pixel overlap detected
Mixed Media Imported images remain visible Photo collage with painting

Using the Plugin

Starting a Session

When you create or open a document in Krita, the plugin automatically begins monitoring your work.

Checking Status

Open Tools → Scripts → Scripter to see debug messages confirming the plugin is active. You should see messages like:

  • CHM Verifier: Setup called
  • CHM Verifier: Loaded CHM library version 0.1.0
  • CHM Verifier: Event capture started

Troubleshooting

Plugin not appearing in list?

  • Verify the .desktop file is in the correct location (pykrita root, not inside plugin folder)
  • Restart Krita completely

"CHM library not available" error?

  • Rebuild the Rust library: cargo build --release
  • Copy to plugin: cp target/release/libchm.dylib krita-plugin/chm_verifier/lib/chm.so
  • Restart Krita

No debug messages in Scripter?

  • Make sure the plugin is enabled in Settings → Configure Krita → Python Plugin Manager
  • Restart Krita after enabling
  • Open Scripter console before creating/opening a document
